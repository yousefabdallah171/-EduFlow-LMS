# Professional Media Upload & Lesson Attachment

This folder tracks the full specification and delivery artifacts for the professional media upload system.

## Documents

- `spec.md`: Feature requirements, user stories, and acceptance criteria
- `plan.md`: Technical implementation plan and architecture direction
- `research.md`: Key technology decisions and tradeoffs
- `data-model.md`: Upload/media entities and status lifecycle
- `architecture.md`: System components and interaction model
- `quickstart.md`: End-to-end validation scenarios for QA
- `contracts/upload-api.md`: Upload session/chunk/complete API contract
- `contracts/lesson-attachment-api.md`: Lesson mapping and bulk attach API contract
- `tasks.md`: Execution checklist with phased implementation tasks

## Current Focus

Start with `tasks.md` Phase 1 and Phase 2, then deliver US1 (reliable high-volume upload queue) as the MVP slice.
