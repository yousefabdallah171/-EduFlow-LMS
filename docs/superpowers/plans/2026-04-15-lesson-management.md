# Lesson Management with Sections Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a hierarchical lesson management system with sections, allowing admins to organize lessons and attach files, while students browse lessons by section and download course materials.

**Architecture:** Backend adds Section model and endpoints for section/lesson CRUD with proper foreign keys. Frontend gets a two-panel admin interface, new lessons overview page showing cards grouped by sections, and updated lesson detail page with attachments at the bottom. Components are reusable (LessonCard, SectionGroup, ResourcesList) following existing patterns.

**Tech Stack:** Node.js 20 + TypeScript 5.4 (backend), React 18.3 + TypeScript 5.4 (frontend), Prisma ORM, PostgreSQL, TanStack React Query

## Tasks

### Task 1: Add Section Model to Database

**Files:**
- Modify: `backend/prisma/schema.prisma`
- Create: `backend/prisma/migrations/<timestamp>_add_sections.sql` (auto-generated by Prisma)

- [ ] **Step 1: Update schema.prisma to add Section model**

Open `backend/prisma/schema.prisma` and add the Section model before the Lesson model:

```prisma
model Section {
  id            String   @id @default(uuid())
  titleEn       String
  titleAr       String
  descriptionEn String?
  descriptionAr String?
  sortOrder     Int      @default(0)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  lessons       Lesson[]
}
```

- [ ] **Step 2: Update Lesson model to add sectionId**

In the Lesson model, add these fields after the existing fields:

```prisma
  sectionId     String?
  section       Section?  @relation(fields: [sectionId], references: [id], onDelete: SetNull)
```

So the Lesson model should look like:

```prisma
model Lesson {
  id              String           @id @default(uuid())
  titleEn         String
  titleAr         String
  descriptionEn   String?
  descriptionAr   String?
  videoHlsPath    String?
  videoStatus     VideoStatus      @default(NONE)
  durationSeconds Int?
  sortOrder       Int              @default(0)
  isPublished     Boolean          @default(false)
  isPreview       Boolean          @default(false)
  dripDays        Int?
  sectionId       String?
  section         Section?         @relation(fields: [sectionId], references: [id], onDelete: SetNull)
  createdAt       DateTime         @default(now())
  updatedAt       DateTime         @updatedAt
  progress        LessonProgress[]
  videoTokens     VideoToken[]
  uploads         VideoUpload[]
  notes           Note[]
  resources       LessonResource[]
}
```

- [ ] **Step 3: Create and run migration**

Run the following command to generate the migration:

```bash
cd backend && npm run prisma migrate dev --name add_sections
```

Expected output: Migration created and database updated. Prisma client regenerated.

- [ ] **Step 4: Verify schema change**

Check that `backend/prisma/schema.prisma` compiles without errors by running:

```bash
cd backend && npm run prisma generate
```

Expected: No errors. Prisma Client types updated.

- [ ] **Step 5: Commit**

```bash
git add backend/prisma/schema.prisma backend/prisma/migrations/
git commit -m "feat: add Section model and link to Lesson"
```
